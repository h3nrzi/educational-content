# Contributing Guidelines

Thank you for contributing to this educational content collection!

## How to Contribute

### Adding New Content
1. Choose the appropriate folder for your content
2. Use the provided template (TEMPLATE.md) for course notes
3. Follow the existing structure and naming conventions
4. Include clear examples and explanations

### Content Guidelines
- Write clear, concise explanations
- Include practical examples
- Add exercises when possible
- Cite sources and give credit
- Use proper markdown formatting

### File Organization
- Use descriptive file names
- Keep related content together
- Add README files for new sections
- Update main README when adding major sections

### Code Standards
- Include comments in code examples
- Use consistent formatting
- Test code before submitting
- Include multiple language examples when relevant

### Submitting Changes
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test your content
5. Submit a pull request

## Content Sources
When adding content from courses or tutorials:
- Always credit the original source
- Don't copy content verbatim
- Summarize and add your own insights
- Include links to original materials

## Questions?
Open an issue if you have questions about contributing!
